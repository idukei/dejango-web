Ataque y Detección MSBuild.exe¶

MSBuild.exe tiene una capacidad integrada llamada "Tareas en línea". Estos son fragmentos de código C# que se pueden usar para enriquecer el proceso de compilación de C#. Lo que esto hace es tomar un archivo XML, compilarlo y ejecutarlo en la memoria en el destino.

Objetivo

Detectar cuando se ejecuta MSBUILD.exe, intentos de omitir la lista blanca, o de realizar un movimiento lateral utilizando herramientas de desarrollo confiables.

Categorización

Evasión de Defensa

Estrategia

  • Supervisar registros de Windows Sysmon, para la creación de procesos.
  • BUscar ejecución de MSBuild.exe
  • Descartarcualquier buen uso.
  • Alerta procesos inusuales de MSBild.exe.

Contexto Técnico

MSBuild es una técnica descubierta por Casey Smith ( @SubTee ) para ejecutar código y eludir el bloqueo de aplicaciones, la protección del dispositivo u otras soluciones de listas blancas. Es ideal para ejecutar cargas útiles de primera etapa y luego realizar técnicas de inyección más avanzadas para implantes (casi) sin disco y C2.

Puntos ciegos y suposiciones

Esta estrategia se basa en los siguientes supuestos:

  • Sysmon se ejecuta y funciona correctamente en el sistema.
  • Los registros de las herramientas de punto final se informan al servidor.
  • Las herramientas de punto final están reenviando correctamente los registros a SIEM.
  • SIEM está indexando con éxito los registros de herramientas de punto final.

Se producirá un punto ciego si se viola alguna de las suposiciones. Por ejemplo, lo siguiente dispararía la alerta:

  • Se cambia el nombre de MSBuild.exe
  • Se abusa de un host incluido en la lista blanca, por ejemplo, un servidor de compilación DevOps
  • Las herramientas de puntos finales se modifican para no recopilar eventos de carga de módulos ni informar al servidor.

Falsos positivos Hay varios casos en los que se producirán falsos positivos:

  • Un usuario realiza legítimamente una compilación durante el desarrollo del software
  • Un proceso automatizado durante las instalaciones de software construye un componente C#

Fuentes

Respuesta

Procedimientos de respuesta:

  • Compare el uso sospechoso de MSBuild con las entradas de la lista blanca.
    • Tenga en cuenta si hay problemas menores debido a las diferencias de ruta o letra de unidad.
  • Compruebe la firma digital del binario.
    • Utilice herramientas o PowerShell para identificar si el binario está firmado digitalmente.
    • Realice una determinación de confianza en el firmante y el binario.
  • Identificar si el binario corresponde a una aplicación instalada.
  • Comportamiento de ejecución del binario.
    • ¿Ha hecho alguna conexión de red inusual?
    • ¿Ha generado procesos secundarios?
    • ¿Ha realizado alguna modificación de archivo sospechosa? Si el binario no es confiable o no se puede rastrear hasta una aplicación instalada legítima, trátelo como un posible compromiso y escale a un incidente de seguridad.
