Ataque y Detección MSBuild.exe¶

MSBuild.exe tiene una capacidad integrada llamada "Tareas en lnea". Estos son fragmentos de cdigo C# que se pueden usar para enriquecer el proceso de compilacin de C#. Lo que esto hace es tomar un archivo XML, compilarlo y ejecutarlo en la memoria en el destino.

Objetivo

Detectar cuando se ejecuta MSBUILD.exe, intentos de omitir la lista blanca, o de realizar un movimiento lateral utilizando herramientas de desarrollo confiables.

Contexto Tcnico

MSBuild es una tcnica descubierta por Casey Smith ( @SubTee ) para ejecutar cdigo y eludir el bloqueo de aplicaciones, la proteccin del dispositivo u otras soluciones de listas blancas. Es ideal para ejecutar cargas tiles de primera etapa y luego realizar tcnicas de inyeccin ms avanzadas para implantes (casi) sin disco y C2.

Puntos ciegos y suposiciones

Esta estrategia se basa en los siguientes supuestos:

  • Sysmon se ejecuta y funciona correctamente en el sistema.
  • Los registros de las herramientas de punto final se informan al servidor.
  • Las herramientas de punto final estn reenviando correctamente los registros a SIEM.
  • SIEM est indexando con xito los registros de herramientas de punto final.

Se producir un punto ciego si se viola alguna de las suposiciones. Por ejemplo, lo siguiente disparara la alerta:

  • Se cambia el nombre de MSBuild.exe
  • Se abusa de un host incluido en la lista blanca, por ejemplo, un servidor de compilacin DevOps
  • Las herramientas de puntos finales se modifican para no recopilar eventos de carga de mdulos ni informar al servidor.

Falsos positivos Hay varios casos en los que se producirn falsos positivos:

  • Un usuario realiza legtimamente una compilacin durante el desarrollo del software
  • Un proceso automatizado durante las instalaciones de software construye un componente C#

Respuesta

Procedimientos de respuesta:

  • Compare el uso sospechoso de MSBuild con las entradas de la lista blanca.
    • Tenga en cuenta si hay problemas menores debido a las diferencias de ruta o letra de unidad.
  • Compruebe la firma digital del binario.
    • Utilice herramientas o PowerShell para identificar si el binario est firmado digitalmente.
    • Realice una determinacin de confianza en el firmante y el binario.
  • Identificar si el binario corresponde a una aplicacin instalada.
  • Comportamiento de ejecucin del binario.
    • Ha hecho alguna conexin de red inusual?
    • Ha generado procesos secundarios?
    • Ha realizado alguna modificacin de archivo sospechosa? Si el binario no es confiable o no se puede rastrear hasta una aplicacin instalada legtima, trtelo como un posible compromiso y escale a un incidente de seguridad.
